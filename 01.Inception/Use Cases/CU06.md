## CU06 – Login del usuario en la app web (Firebase Auth)

**Nombre**  
CU06 – Login en la app web

**Actor principal**  
Usuario (nuevo o existente).

**Objetivo**  
Iniciar sesión en la app web de Prompt Library para que todo (prompts, XP, etc.) quede asociado a su identidad.

**Precondiciones**

- App web accesible (Next.js en Vercel).  
- Proyecto de Firebase creado con al menos un método de login activo (ej. Google o email/password). [zestminds](https://www.zestminds.com/blog/supabase-auth-nextjs-setup-guide/)

**Flujo principal**

1. El usuario entra en la web (ej. `/` o `/login`).  
2. Ve un botón de “Iniciar sesión” (por ejemplo “Continuar con Google” o email/password).  
3. El usuario pulsa el botón de login.  
4. El frontend usa Firebase Auth para iniciar sesión (ej. `signInWithPopup` con Google o `signInWithEmailAndPassword`). [firebase.google](https://firebase.google.com/codelabs/firebase-nextjs)
5. Firebase devuelve un `user` autenticado y un ID token.  
6. El frontend guarda el estado de sesión (contexto/estado global) y, opcionalmente, llama a tu backend para registrar/actualizar el perfil del usuario (uid, email). [reddit](https://www.reddit.com/r/nextjs/comments/19dbiue/best_practice_nextjs_auth_firebase/)
7. El usuario es redirigido al dashboard (lista de prompts).

**Postcondiciones**

- La app conoce el `uid` del usuario de Firebase.  
- La extensión podrá reutilizar ese mismo login (mismo proyecto Firebase) para firmar peticiones a tu API.

## CU07 – Login en la extensión (reusar Firebase Auth)

**Nombre**  
CU07 – Login en la extensión de navegador

**Actor principal**  
Usuario que quiere guardar/usar prompts desde el navegador.

**Objetivo**  
Autenticar al usuario en la extensión usando el mismo Firebase Auth que la app web, para que las acciones de la extensión se asocien a su cuenta.

**Precondiciones**

- Extensión instalada.  
- Mismo proyecto de Firebase configurado en la extensión. [firebase.google](https://firebase.google.com/docs/auth/web/chrome-extension)

**Flujo principal (ejemplo Google login)**

1. El usuario abre el popup de la extensión (icono en la barra) o una acción que requiera login (guardar/usar prompt).  
2. Si no hay sesión, el popup muestra “Inicia sesión para usar la librería” con botón “Login con Google”.  
3. El usuario pulsa el botón.  
4. La extensión llama a Firebase Auth especializado para extensión (Manifest V3) usando offscreen / iframe según la guía oficial para Chrome extensions. [dev](https://dev.to/lvn1/google-authentication-in-a-chrome-extension-with-firebase-2bmo)
5. El usuario completa el flujo de Google en la ventana emergente.  
6. Firebase devuelve un usuario autenticado y su ID token.  
7. La extensión guarda esos datos en `chrome.storage` y marca la sesión como activa.  
8. A partir de ese momento, todas las llamadas a tu API (`/api/prompts`, `/api/prompts/use`, etc.) enviarán el ID token en las cabeceras para que el backend pueda identificar al usuario.

**Postcondiciones**

- Extensión y web comparten la misma identidad (mismo Firebase uid).  
- Tu backend puede mapear `uid` → `user_id` en Supabase y aplicar RLS, XP, etc. [supabase](https://supabase.com/docs/guides/auth/third-party/firebase-auth)
